# Outcome

管理员打开「服务器与节点」后能在合理时间内看到服务器列表（成功或明确失败），不再无限显示「正在读取服务器…」。探针全景大盘（含地图）在公开接口短暂失败后仍能恢复展示已有服务器，不因一次挂起请求而永久空白。

# Scope

- 管理端舰队概览 `loadFleetServers` 的请求超时、错误展示与加载态收口。
- 探针公开列表 `fetchProbeData` 的超时、失败时保留已有数据、必要时重试。
- 实时通道已连接但公开列表为空时，不得永久停止探针列表回源。
- 地图在 `publicProbeServers` 有数据时应能画出标记；对 geojson 拉取失败给出可诊断行为（至少不静默依赖失败状态）。

# Non-goals

- 不重做舰队三层信息架构与分页 API 契约。
- 不改变 Agent 上报语义、鉴权或 D1 schema。
- 不扩大 VPS 上限，不改探针主题/公开策略产品语义。
- 不保证 Worker 在 Cloudflare 侧 100% 零超时；保证前端不永久转圈、可恢复。

# Acceptance examples

1. **舰队概览成功读取**  
   - 输入：管理员登录后进入「服务器与节点」；`GET /api/servers` 在超时上界内返回至少一台机器。  
   - 输出：列表展示返回的服务器；「正在读取服务器…」消失。

2. **舰队概览读超时**  
   - 输入：管理员在服务器概览触发刷新；`GET /api/servers` 超过客户端超时仍无完整响应。  
   - 输出：不再显示「正在读取服务器…」；展示可读错误；用户可再点「刷新」重试。

3. **探针列表可恢复**  
   - 输入：`GET /api/probe/public` 首次超时或失败，随后成功；或实时通道已连接但公开列表仍为空。  
   - 输出：地图/卡片最终能出现服务器（经重试或回源），不会永久空白。

4. **探针失败保留缓存**  
   - 输入：公开列表已有服务器数据；后续 ajax 探针公开读失败。  
   - 输出：保留上一份服务器列表与国家统计；地图不因该次失败被清空。

# Constraints and invariants

- 列表接口仍是 `GET /api/servers` / `GET /api/probe/public`；不改响应字段契约。
- 不得在列表响应中暴露 agent token / 私钥 / 密码。
- 超时时间对用户可见操作应有上界（默认约 12–20 秒量级），避免无限 pending。
- Abort 与并发刷新不得留下 `fleetServersLoading === true` 且无进行中请求的死态。

# Decisions

- 客户端对舰队与探针公开读请求加超时；超时视为失败并清理 loading。
- 探针公开读失败时：若已有缓存数据则保留；空列表时允许在实时连接下继续有限回源/重试。
- 不在本 change 中大改后端 Cache API 或 Agent 上报路径（除非前端修复后仍必现，再另开）。
