# subscription-export

## Purpose

定义用户客户端订阅导出（`GET` 订阅接口）在节点集合、格式与凭据暴露方面的完整目标行为。本 capability 约束「订阅里会出现什么」，不定义住宅代理控制面或 VPS 出口编排。

## Formats

订阅接口在鉴权通过后提供两种用户可见格式：

1. **普通订阅（默认）**  
   节点以代理链接列表形式给出（实现上可对整表做 Base64 封装，与现有客户端约定一致）。

2. **Clash 订阅**  
   当请求带 Clash 格式参数时，返回可被 Mihomo / Clash Meta 消费的 YAML，至少包含 `proxies`、代理组与既有规则/rule-providers 装配语义。

两种格式共享同一套「可导出节点集合」规则；差异仅在序列化形态。

## Authentication and eligibility

- 请求必须携带有效的用户（或管理员）订阅身份与 token；无效时表现为资源不存在类响应，不泄露节点。
- 若启用订阅保护策略，在校验前可返回无害页面响应（既有保护语义保留）。
- 可导出的协议节点须满足：启用、未超额、未过期，并符合该用户/管理员既有可见范围（含可选按 VPS IP 过滤）。
- 已启用的第三方节点可并入同一订阅集合（既有整合语义保留）。

## Exportable node set

订阅可包含：

- 上述资格内的**协议节点**（含用户或管理员创建的协议型 Socks5 等）；
- 已启用的**第三方订阅节点**。

订阅**不得**包含：

- 来自住宅代理运行时登记（`proxy_ctrl_servers` 或等价运行时住宅记录）的 **住宅 SOCKS5** 条目；
- 为上述住宅条目写入的共享凭据（环境中的住宅 `PROXY_USER` / `PROXY_PASS` 或同等密钥）出现在链接、YAML `username`/`password` 或备注中的导出。

管理员订阅与普通用户订阅在「是否导出住宅 SOCKS5」上行为一致：**一律不导出**。

## Credentials and safety

- 协议节点凭据按其协议既有字段导出（uuid、私钥等），不因本 capability 扩大暴露面。
- 住宅代理共享凭据不得进入任一订阅格式的响应体。
- 验证与运维文档不得复述真实凭据值。

## Out of scope for this capability

- 住宅控制面列表/上报接口（供 Agent 或外部控制器拉取 SOCKS5 列表）的存在与鉴权模型。
- VPS 上将流量走住宅出口的 egress 配置与本机 SOCKS 监听。
- Clash 规则正文、rule-providers 模板内容的业务含义（仅要求装配路径不被本变更破坏）。
- 订阅 token 的颁发与轮换流程。

## Acceptance criteria

1. 管理员普通订阅与 Clash 订阅均不含住宅 SOCKS5 追加条目及住宅共享凭据。
2. 普通用户普通订阅与 Clash 订阅均不含住宅 SOCKS5 追加条目及住宅共享凭据。
3. 仅有活跃住宅记录、无可导出协议/第三方节点时，订阅不因住宅记录而出现 SOCKS5 节点。
4. 协议表中的 Socks5 节点仍可按既有规则出现在适用格式中。
5. 住宅控制面接口不因「订阅去暴露」而被要求删除（可继续服务非订阅调用方）。
