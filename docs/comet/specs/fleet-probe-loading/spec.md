# fleet-probe-loading

## Purpose

定义管理端服务器概览与探针公开大盘在**读路径失败或超时**时的完整目标行为，保证用户可见的加载态可结束、列表可恢复，地图在有数据时能展示服务器。

## Definitions

| 术语 | 含义 |
| --- | --- |
| 舰队概览读 | 管理员在「服务器与节点」服务器概览使用的分页读：`GET /api/servers` |
| 探针公开读 | 探针全景使用的 `GET /api/probe/public`（含 `ajax=1`） |
| 读超时 | 客户端在约定上界内未收到完整响应，主动中止请求并记为失败 |
| 缓存列表 | 前端内存中已成功加载过的 `fleetServers` / `publicProbeServers` |

## Requirements

### 舰队概览加载

1. 管理员进入服务器概览时发起舰队概览读；成功时展示返回的服务器页并清除加载态。
2. 舰队概览读必须带**有限超时**（实现默认不超过 20 秒）。超时或网络错误时：
   - 不得继续显示「正在读取服务器…」作为唯一状态；
   - 应展示可读错误文案；
   - 用户可通过刷新按钮再次发起读。
3. 并发刷新或筛选变更导致中止旧请求时，最终只保留最新一次请求的结果与 loading 状态；不得留下「无进行中请求且 loading 永久为 true」。
4. HTTP 非 2xx 仍通过既有鉴权/错误路径处理；401 保持登出语义。

### 探针公开列表与地图

5. 探针公开读必须带有限超时；超时视为失败。
6. 探针公开读**失败且已有缓存列表**时，保留缓存列表与由它派生的国家统计/地图标记，不得因一次失败清空为「无服务器」。
7. 探针公开读**失败且缓存为空**时，允许有限重试或在实时通道已连接时仍做回源，不得因「实时已连接」永久停止列表拉取导致地图永久无点。
8. 地图标记仍由 `publicProbeServers` 的国家字段聚合生成；有服务器且国家码可映射时，地图上应出现对应标记（geojson 底图拉取失败时，至少不因列表成功而阻塞标记逻辑的可诊断重试路径）。

### 不变项

9. 不改变 `/api/servers` 与 `/api/probe/public` 的成功响应字段契约。
10. 不在列表响应或前端日志中输出凭据、agent token、节点私钥。

## Non-requirements

- 不消除 Cloudflare Worker / D1 侧所有偶发超时。
- 不重写 Cache API 或 Agent `/api/report` 热路径（可后续独立优化）。
- 不改变舰队分页、筛选、单机工作台写操作语义。

## Acceptance criteria

1. **舰队概览成功**  
   - 输入：管理员打开服务器概览且 `/api/servers` 成功返回非空页。  
   - 输出：展示服务器行；加载态关闭。

2. **舰队概览超时**  
   - 输入：舰队概览读超时。  
   - 输出：加载态关闭；出现错误文案；可再次刷新。

3. **探针失败保留**  
   - 输入：已有 `publicProbeServers` 缓存时公开读失败。  
   - 输出：列表与地图统计不因失败清空。

4. **探针空列表恢复**  
   - 输入：列表为空且实时已连接或首次公开读失败。  
   - 输出：仍允许有限回源/重试直至成功或给出失败状态，不得永久空白无重试。
