# Vision：KUI x Server Monitor Pro

## 产品核心

KUI 为**自托管运维者**提供一套「装完就能管」的目标体验：在**单一 Cloudflare Worker** 上同时完成代理节点舰队管理、服务器探针大盘与多用户配额分发。

最小本质：**不用自建传统面板服务器**，打开 Worker 域名即可部署节点、分发订阅、盯机器是否活着。

服务三类结果：

1. **运维者**能把 VPS 变成可管理、可订阅的协议节点，并控制谁用、用多少。
2. **终端用户**能拿到可用的普通 / Clash（Mihomo）订阅并上网。
3. **观众**能通过公开探针页看到机房与节点健康画像（可定制外观）。

## 用户旅程

主阅读路径按角色，不按代码目录。

### 运维者（主用户）

- **一键上线面板**：Deploy to Cloudflare Workers 或本地 `wrangler deploy` → 首次访问初始化 D1 → 换掉默认 `ADMIN_PASSWORD` → 进入管理端。
- **把机器变成节点**：服务器与节点 → 添加 VPS 名称与公网 IP → 复制 Full Deploy Command（root 执行）或 8 合 1 批量下发 → Agent 回连 → 创建协议节点。
- **给人开权限、控流量**：多用户管理 → 配额 / 到期 / 订阅令牌 / 启停。
- **看机房与节点健康**：探针全景大盘 + 实时 WebSocket + Cron 离线检查；可选 Telegram 告警。
- **换出口 / 中转 / 外挂**：住宅 IP 代理、Realm 中转、第三方服务与订阅、系统设置（Realtime 频率、探针主题、代理参数等）。

### 终端用户（被开通者）

- **拿订阅用客户端**：「我的主页」→ 复制普通订阅 / Clash 订阅 / 二维码 → 在 Mihomo、Clash 等客户端导入。
- **看自己开了什么**：已开通节点列表；按权限使用订阅保护与令牌重置（运维策略决定）。

### 观众

- **看公开探针**：公开探针主题页（卡片 / 表格 / 地图等）→ 实时状态与展示信息；运维可配主题、CSS、背景。

```text
                    ┌──────────── 运维控制台 ────────────┐
部署与平台 ──► 准入与账号 ──► 服务器与节点 ──► 节点出口与中转
     │              │              │                    │
     │              └── 多用户与订阅 ──► 订阅导出 ───────┘
     │                              │
     └──────── 实时与探针大盘 ◄──────┘
                    │
              通知与扩展（TG / 第三方 / WebSSH / AI 中转…）
```

| 产品区域 | 支撑哪些旅程 | 与谁协作 |
|---|---|---|
| **部署与平台** | 一键上线、本地发布、自定义域名 | 为全部区域提供运行面（Worker Assets、D1、内置 DO 实时） |
| **准入与账号** | 管理员登录、账号安全、默认 Secret 治理 | 保护控制台与管理 API |
| **服务器与节点** | VPS 接入、Agent、多协议节点、批量下发 | 喂给订阅导出、探针、出口 |
| **多用户与订阅** | 开通用户、配额、到期、令牌、订阅保护 | 决定谁能拉订阅、能用多久 |
| **订阅导出** | 普通订阅 / Mihomo·Clash（含 AnyTLS） | 终端用户主出口 |
| **节点出口** | 原生、WARP、住宅代理、手动 SOCKS5 | 改变节点实际出网路径 |
| **中转与扩展** | Realm、第三方服务/订阅、WebSSH、AI 中转、Proxy Controller 等 | 扩展运维工具箱；与核心舰队协作但可降级理解 |
| **探针与实时** | CPU/内存/盘/网/延迟、主题、公开页、观众频率自适应 | 运维盯盘 + 观众页 |
| **通知** | Telegram 告警、Cron 离线检查 | 挂在探针与节点在线状态上 |

## 基础与边界

- **运行形态（目标）**：保持「单 Worker 自托管」；前端与 VPS 安装组件走 Worker Assets；实时 WebSocket 内置于主 Worker（不依赖独立 Realtime Worker）。
- **数据边界**：配置、用户、节点、流量、探针等进 D1（binding 名 `DB`）；实时在线状态与 Dashboard Hub 进 Durable Objects。
- **信息安全性（Security）**：一键部署默认弱口令与默认代理凭据仅便于装机；公开使用前必须覆盖 `ADMIN_PASSWORD`、`PROXY_USER`、`PROXY_PASS` 等 Secret；自定义密钥、Token、D1 ID 不上仓库。
- **平台边界（当前目标）**：以 Cloudflare Workers + D1 + Durable Objects 为第一等运行面；非 CF 运行时不在本 Vision 主路径。
- **产品边界（候选，见探索空间）**：是否长期单管理员 + 多终端用户；扩展菜单是否一律算核心愿景。

## 探索空间

以下方向在对话中标为未拍板，**意图状态为候选 / 探索中**，不假装已收束。

- **产品重心** `[探索中]`：代理舰队为主 / 探针监控为主 / **两者并重**。当前能力版图按并重地图书写；若你确认偏重一侧，应重排旅程与版图权重。
- **扩展区定位** `[候选]`：Realm、第三方服务/订阅、WebSSH、AI 中转、ByteVirt、Proxy Controller 等——核心愿景内 vs 周边可选插件区。当前按「中转与扩展」区域收录，未强制全部为核心。
- **组织与权限** `[候选]`：长期单管理员 + 多终端用户 vs 多管理员 / 组织级权限。
- **明确不做** `[探索中]`：多租户 SaaS、非 Cloudflare 主路径部署等——尚未书面封口；需要时可升格为边界。

## 统一语言

| 术语 | 含义 | 不要和哪些词混用 |
|---|---|---|
| **运维者** | 部署并管理 KUI 的人 | ≠ 终端用户、观众 |
| **终端用户** | 被开通、主要消费订阅的人 | ≠ 运维者、观众 |
| **观众** | 只看公开探针、不持管理权的人 | ≠ 终端用户 |
| **节点** | 可被订阅/代理使用的协议入口 | ≠ 仅指 VPS 主机 |
| **VPS / 机器** | 跑 Agent 的主机；一台可挂多节点 | ≠ 节点 |
| **Agent** | VPS 上的接入与上报组件 | ≠ 面板后端 |
| **探针** | 机器与链路观测与展示 | ≠ 单独的代理连通性测试 |
| **订阅** | 给客户端的节点清单与访问令牌 | ≠ 面板登录会话 |
| **实时** | 主 Worker 内 DO + WebSocket 通路 | ≠ 已废弃/不必再配的独立 Realtime Worker |
| **信息安全性（Security）** | 防未授权访问与篡改 | ≠ 安全性（Safety，本产品主路径一般不强调） |
